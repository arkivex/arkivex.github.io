Concept

Arkivex provides an archive service that stores and manages documents and data. It offers a single place where important data or documents from many systems can be collected, connected, stored and preserved. It then provides a single point where applications and people can access this data.

Applications can be configured to deliver to the archive based on metadata or events. Out of the box integrations to Office 365, shared file systems and more, combined with a great API and ready to use client SDKs, means that organisations can be up and running in moments and develop integrations with all existing and new systems, or cloud services.

A great management studio is provided to allow archive administrators to curate the archive and manage different aspects such as defining policies for retention, managing controlled vocabularies, and management of users

For end users there is a simple but powerul search interface that allows them to search and navigate the archive. One of the unique things of Arkivex is its true point-in-time query ability. This allows users to view the complete archive including metadata links between artifacts at a given point in time.

A flexible data model allows for compliance with different standards and industry specific vocabularies. Arkivex data objects are schemaless, but can be validated and checked against different metadata schemas. Exports in common standards such as Noark5 and Moreq2010 are also supported.

Features

Sign In with Business Accounts

Users can sign-in with Office365, Google or Github accounts.

Manage Users and Roles

Define which users can do what via the management studio.

Office365 Connector

Easy to Configure connector that can synchronise content from Office365/SharePoint/OneDrive based on folders or metadata.

Google Docs Connector

Easy to Configure connector that can synchronise content from Google Docs based on folders or metadata.

Shared Folder Connector

Simple connector that can synchronise a shared folder. *requires local running of code for on-premise share.

Automatic Conversion of Office Docs to PDF/1A

In addition to keeping the original document, converts office docs to PDF/1A - the standard for long term storage.

Great Search UI

Simple and powerful UI for searching and navigating the archive.

Support for Continuous Delivery

To the archive via APIs, and from the archive APIs expose the changes allowing clients to pull and store them.

Standards Based Exports

Arkivex Simple Archive format, Noark5 and Moreq2010. Also, as JSON, XML and RDF.

RESTful API

Easy to use, well documented RESTful api for managing archives.

Client SDKs

Ready to use, client APIs for dotnet, and javascript that call to the REST api.

Powerful and Extensible Metadata Model

Create extensions to the core archive model, create controlled vocabularies, connect data and documents into a unified graph of information. Search the entire graph at any point in time.
